## 1. What the product is

StatementAudit Pro is a React/Vite SPA + Express server that:
1. Accepts PDF bank statements from any UK bank (drag-drop, up to 50 at once)
2. Extracts transactions using Claude (AI) + a deterministic text-layer extractor (dual-path, cross-checked)
3. Reconciles extracted transactions against the printed opening/closing balances
4. Presents transactions for inline human review and coding
5. Exports coded CSVs for Xero (pre-coded via Code & Create) or QBO (reference via Code & Reference)
6. Produces an Audit Workbook (3-sheet Excel) for professional sign-off

## 3. Features shipped (complete as of 2026-06-30)

### Core parsing
- AI extraction via claude-sonnet-4-6 (model pinned in source at line 468 and in verify.sh)
- Dual-path extraction: LLM + deterministic text-layer (parallel, cross-checked)
- Cross-check statuses: agree / partial / count_mismatch / unavailable
- Automatic reconciliation against opening/closing balances
- Re-run on any statement at any time

### Account & jurisdiction
- Account types: Current, Savings, Credit Card, Business, ISA/Cash ISA, Mortgage
- Tax jurisdictions: UK VAT (20%/5%/Zero/Exempt/Outside Scope) · Jersey GST (5%/Zero/Exempt/ISE/Outside Scope) · Other (no tax column)

### Review
- Inline cell editing — amounts, dates, descriptions, categories
- Editable opening/closing balance override
- ⚑ uncertainty flags on AI-uncertain rows
- 💱 FX badge + free live rate lookup (reference only; never used in arithmetic)
- Receipt attachments (PDF/image per transaction row)
- Roll back Approved → Review at any time
- Keyboard shortcuts: A=Approve, R=Reject, ←→=navigate, ?=shortcuts

### Confidence scoring
- 7-factor granular score 0–100 replacing binary pass/fail
- Tiers: ⚡ High (95+) · Good (80–94) · Fair (70–79) · Review (<70)
- Hover badge shows one-line explanation of what to fix

### Coding — Xero Pathway 2 (Code & Create)
- Confirm code per line → precoded CSV with Account Code + Tax Rate + Tracking1/2
- Empty-period hard gate (checkbox assertion before export activates)
- Payee/description editable before confirm
- Chart of accounts autocomplete (import CSV or pull live from Xero OAuth)
- Tracking categories (import CSV; up to 2 per line; written to precoded export)
- Auto-confirm remembered payees switch

### Coding — QBO Pathway 1 (Code & Reference)
- 9-column QBO CSV (Date/PaymentType/Description/Payee/Debit/Credit/Category/NominalCode/Notes)
- Chart of accounts via CSV import or live QBO OAuth pull
- Codes written as reference columns; human applies in QBO

### Layer 2 AI suggestions
- Background AI lookup for unknown payees → purple ✦ suggested code
- Human ✓ confirmation required; no auto-apply path ever
- ✦ disappears once line confirmed

### Payee memory
- Code, category, GST/VAT treatment, tracking remembered per payee
- Persists across sessions; shared in workspace

### Projects
- Multi-client organisation; every statement belongs to a project
- ◈ Projects dashboard: approved/pending/error counts + last activity per project
- Create, rename, delete projects

### Audit Workbook
- 3-sheet Excel: (1) Full transaction register with all fields; (2) Clean import data; (3) Receipt index
- Merge & Export: combine all approved statements from a project into one file
- Compatible with Excel Copilot and Google Sheets Gemini for AI-assisted review

### Cloud & Practice Workspace
- Google Drive: personal cloud save to private app folder
- Microsoft OneDrive: personal cloud save
- M365 Practice Workspace: shared OneDrive folder; payee memory/CoA/tracking shared across team (scope: Files.ReadWrite)
- Zero server-side storage: financial data never passes through Render; Drive/OneDrive is the storage layer

### UX
- Guide Mode: hover tooltips on every control (persists between sessions)
- In-app searchable Help panel (all features documented)
- Backup/restore payee rules as .json
- Batch 50: up to 50 PDFs per session

## 4. Security & compliance constraints (non-negotiable)

These constraints must not be changed without explicit written instruction:

| Constraint | Rule |
|---|---|
| Human approval gate | Only path to any export — including Xero Pathway 2. No auto-post to any ledger, ever. |
| AI role | Model transcribes; deterministic code does all arithmetic. Coding stays a deterministic lookup, not model-inferred at Layer 1. |
| API key | Server-side only; never exposed to browser or committed to repo |
| Layer 2 suggestions | Proposals only; human ✓ required; no auto-apply path |
| FX rate | Reference only; never used in arithmetic or written to any CSV |
| CoA tokens | Xero/QBO OAuth tokens must NOT be written to localStorage, sessionStorage, or persisted React state |
| Pathway 2 scope | Scoped to empty periods only (Xero) — the empty-period assertion is a hard gate |
| Compliance gate | No real **client** data processed (free or paid) until compliance gate is closed — processing a client's data makes the app a data processor regardless of payment, which is exactly what JOIC + DPAs cover |
| GST/VAT arithmetic | gstJersey/vatUK MUST NOT be referenced inside recalc, the balance walk, reconciliation arithmetic, or BASE_PROMPT/PROMPTS |
| QBO reference | The _CODED_REF.csv must not imply or claim QBO applies codes automatically |

## 6. Beta professional user test

### Objective
Test StatementAudit Pro end-to-end with 5 practising UK accounting professionals using their own real statements (with consent). Gather structured feedback to identify reliability issues, UX friction, and workflow gaps before public launch.

### User profile criteria
Participants should match one or more of these profiles:
- Bookkeeper or accountant in UK practice (sole trader or small firm)
- Uses Xero or QBO as their primary ledger
- Processes bank statements for clients regularly
- Comfortable with browser-based tools

### Participant agreement — what to include
Each user should confirm in writing (email or message is sufficient at beta stage) that they:
1. **Consent to process their own or client statements** — confirming they have authority to use the data in a testing context
2. **Understand the app is beta** — results may not be 100% accurate; human review is required before any data is relied upon
3. **Will not submit real client data to a third-party AI without informing the client** — or alternatively, use their own practice account statements for testing
4. **Will provide honest feedback** on accuracy, usability, and any issues found
5. **Will not share the app URL publicly** during the beta period

Template wording:
> "I confirm I have authority to process the statements I am testing, I understand StatementAudit Pro is in beta, and I agree to provide feedback on my experience. I will not share the app link externally during the beta period."

### Test tasks per user

Each participant should complete the following workflow:

**Task 1 — Upload and process (20 mins)**
- Upload 3–5 real bank statement PDFs (own or consented client statements)
- Set correct Account Type and Tax Jurisdiction for each
- Run Process All
- Review extracted transactions; note any errors or missed transactions

**Task 2 — Code and export (20 mins)**
- Open Code & Create (Xero) or Code & Reference (QBO) for at least one statement
- Import their chart of accounts CSV (or use QBO/Xero OAuth pull)
- Confirm codes for all transactions
- Export the coded CSV
- Optionally: import the CSV into their Xero/QBO sandbox and report whether it lands correctly

**Task 3 — Audit Workbook (10 mins)**
- Download the Audit Workbook for processed statements
- Open in Excel or Google Sheets
- Check the transaction register for accuracy
- Optionally: try an Excel Copilot or Gemini prompt from the in-app Help guide

**Task 4 — Free exploration (10 mins)**
- Use Guide Mode to explore features they haven't tried
- Note any flows that felt unclear or missing

### Feedback collection

After completing tasks, ask each user to answer:

1. **Accuracy** — What percentage of transactions extracted correctly on first pass? (estimate)
2. **Reconciliation** — Did all statements reconcile (green)? If not, what was the issue?
3. **Coding workflow** — Was Code & Create / Code & Reference clear? Any friction?
4. **Audit Workbook** — Useful? Anything missing?
5. **Biggest issue encountered** (free text)
6. **Feature most useful for their workflow** (free text)
7. **Feature missing that would make this a daily tool** (free text)
8. **Net Promoter score** — On a scale of 0–10, how likely are you to recommend this to a colleague?

### Success criteria

| Metric | Minimum pass threshold |
|---|---|
| Reconciliation pass rate | ≥ 80% of statements reconcile without manual balance override |
| Transaction extraction accuracy | ≥ 95% of transactions extracted correctly (count + amounts) across all tested statements |
| Duplicate detection | No participant reports a missed duplicate that was visible in the statement |
| Coding export | CSV imports into Xero/QBO without mapping errors for ≥ 4 of 5 participants |
| NPS | Average score ≥ 7 |
